# Mafia Client

Веб-клієнт онлайн-гри **«Мафія»** на React + TypeScript. Відмальовує головну
сторінку, лобі, кімнати та ігровий екран із голосовим/відеочатом, а всю логіку
гри бере від [`mafia-server`](../mafia-server) — сервер є джерелом істини, клієнт
лише показує стан і шле дії гравця.

> Дипломний проєкт. Стек: React 19, Vite 7, React Router 7, Tailwind CSS v4,
> LiveKit (голос/відео), Vitest.

## Можливості

- **Вхід за nickname** без пароля; сесія зберігається у `sessionStorage`.
- **Лобі та кімнати**: список, створення, приєднання, старт гри власником.
- **Ігровий екран**: фази, кроки, таймери (синхронізовані з годинником сервера),
  ролі, дії, журнал подій.
- **Ознайомчий перший раунд** з відповідними підказками (без вибору цілі/голосування).
- **Голосовий і відеочат** через LiveKit, керований фазами гри.
- **Таємниця нічних камер**: вночі камери учасника бачать лише ті, кому дозволено
  (напр. мафія — одне одного), решта бачить аватар.
- **Візуалізація голосування** в стилі Among Us: на тайлі гравця з'являються
  іконки тих, хто за нього голосує.
- **Лічильник живих/усього** гравців у шапці.
- **Екран завершення** з підсумками ролей і кнопкою вимкнення мікрофона.
- **Realtime** через WebSocket + автоматичне відновлення після reload/reconnect.

## Швидкий старт

**Вимоги:** Node.js **22+**.

```bash
# 1. Клонувати репозиторій
git clone https://github.com/sod7m/mafia-client.git
cd mafia-client

# 2. Встановити залежності
npm install

# 3. Запустити dev-сервер
npm run dev
```

Клієнт підніметься на `http://localhost:5173`. За замовчуванням він ходить у
backend на `http://localhost:8080`, тож **спершу запусти
[`mafia-server`](../mafia-server)** (`go run ./cmd/api`).

## Конфігурація

Адреса бекенду задається через змінну середовища `VITE_API_BASE_URL`
(якщо не задано — `http://localhost:8080`). З неї ж виводиться адреса WebSocket.

```bash
# Тимчасово для dev
VITE_API_BASE_URL=http://localhost:8080 npm run dev
```

Для продакшн-збірки значення береться з файлу `.env.production`:

```env
VITE_API_BASE_URL=https://<ваш-бекенд>.herokuapp.com
```

## Маршрути

| Шлях | Сторінка | Доступ |
|---|---|---|
| `/` | Головна + вхід за nickname | публічний |
| `/rules` | Правила гри | публічний |
| `/rooms` | Лобі (доступні кімнати) | потрібна сесія |
| `/room/:id` | Кімната (склад, код, старт) | потрібна сесія |
| `/room/:id/game` | Ігровий екран | потрібна сесія |
| `/404`, `*` | Сторінка «не знайдено» | публічний |

Закриті маршрути обгорнуті в `ProtectedRoute` — без активної сесії редірект на головну.

## Як це працює

- **Автентифікація.** Вхід лише за nickname (без пароля). Пара `token` + `user`
  зберігається в `sessionStorage`; на сервері токен живе 24 години.
- **Глобальний стан.** `src/context/GameContext.tsx` тримає кімнати, поточну гру,
  WebSocket-підписку і recovery після перезавантаження сторінки.
- **HTTP + Realtime.** Усі запити — через `src/lib/api.ts`. Клієнт слухає `/ws`;
  отримавши сигнал `game.updated`, дочитує **приватний** стан гри через
  `GET /api/games/{roomId}` (сервер віддає кожному лише те, що йому дозволено бачити).
- **Ігровий екран.** `src/pages/GamePage.tsx` рендерить серверний snapshot:
  фази, кроки, таймери (звірені з годинником сервера через заголовок `Date`),
  ролі, дії, журнал подій, тайли гравців із відео/аватарами.

### Ознайомчий перший раунд

Перший раунд — ознайомчий: на ночі ролі лише прокидаються (дії заблоковано), а
на дні немає голосування. Тому на round 1 екран ховає вибір цілі, показує
підказки, а кнопка наприкінці першого дня веде «До ночі». Повноцінна гра — з раунду 2.

## Голос і відео (LiveKit)

Медіа йде через LiveKit Cloud; токен доступу клієнт бере в бекенду
(`POST /api/games/{roomId}/voice-token`). Уся логіка — у `src/context/VoiceContext.tsx`.

Модель керування зроблена «ультимативною» і непробивною:

- **Ефективний мікрофон = `дозвіл системи` AND `бажання користувача`.**
  Гра вмикає/вимикає мікрофон за фазою (говорить лише той, чия зараз черга), але
  якщо користувач сам себе замутив — система **не зможе** його ввімкнути. Натиснути
  на замучений мікрофон і «прорватися» говорити поза своїм ходом неможливо.
- **Камера керується лише користувачем** — система її не вмикає/вимикає примусово,
  тільки **ховає** від тих, кому не можна бачити.
- **Таємниця камер.** Видимість опублікованих доріжок задається дозволами підписки
  LiveKit (`setTrackSubscriptionPermissions`): вночі камеру учасника бачать лише
  дозволені (напр. мафія бачить мафію), а на тайлах інших гравців показується аватар.

Фазову політику звуку/видимості рахує `GamePage.tsx` (`computeAudioPolicy`) і
передає у `VoiceContext` через `setGameAudioPolicy`. Наприклад: на фінальному
екрані говорити можуть усі (зокрема мертві), а вдень — лише активний промовець.

## Деплой

Клієнт — це статичний SPA, що збирається у `./dist`.

- Хоститься на **Cloudflare** (Workers static assets, конфіг `wrangler.jsonc`):
  невідомі шляхи віддають `index.html` (`single-page-application` routing).
- Адреса бекенду на проді береться з `.env.production` (`VITE_API_BASE_URL`).
- Збірка: `npm run build` → задеплоїти вміст `./dist`.

Бекенд деплоїться окремо — див. [`mafia-server`](../mafia-server).
